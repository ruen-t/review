Guidelines

  1. Before requesting for a review through the EasyReview system, make sure that there is no other review on the same timeslot. If there is another review on the same timeslot, please confirm the availability of reviewers by asking them directly. If there are available reviewers, you may register the review. You can check the schedule of reviews through this Menu/Review.
  2. EasyReview notifies reviewers (leader, CIC, QC-Corner, etc.) and managers (PM, PDM, etc.) via e-mail. Make sure to write the reviewers and managers properly when you request for a review.
  3. Ideally, you should request for a review 7 days before the target date. In addition, you must submit materials or documents for review at least three days before the review date. If an “emergency review” is required, please consult with the Reviewers: Morishita, Fukuda, Abe, Nishi, Ishizaki, Honda, Akanuma, Nakamura (Kurobe), Hoshina, Takase, Hase.
  4. If you want to request for RR, you do not need to input project details. If you need to indicate information like SHOP, write it on the title or in the comments section. After you received Development Number, update your review page by indicating the Project Name, Development Number and SHOP.
  5. If you want to request for PDR, CDR, PQR and PSR, be sure to specify the project details like Project Name, Development Number and SHOP.
  6. Specify the Development Type as either of the following:
    1. NEW (e.g. New Menu, Driver Development)
    2. VUP (Version Up if work is more than 1 person for one month)
    3. MAINTE (Maintenance if work is less than 1 person for one month)
  7. If you want to request for PSR, you may only choose from the following timeslots.
    1. 10:00 - 11:00 JST (except Mondays)
    2. 14:00 - 15:00 JST
    3. 16:00 - 17:00 JST (for WITH only)
      (In cases of urgent or critical change control, the PM or PDM may request for other schedules aside from (a), (b) and (c).)
  8. For PSR, be sure to follow the  template for minutes of the meeting in this link. Aside from uploading the minutes on the EasyReview system, you must email the minutes to review@wni.com, cic@wni.com and qc-corner@wni.com.

 

Entry

  1. Fill in the form in the following Menu/Review -> ADD .
  2. For title, use the following format: <Development Item> [<Development Type>]. Example “EasyReview Apollo V.1.2.1 [VUP].”

ガイドライン

  1. EasyReviewシステムでレビューの申請をする前に、同じ時間帯に他のレビューが無いことを確認してください。同じ時間帯にレビューを申請する場合は、直接レビュアーに参加可能かどうか確認してください。また、もしレビュアーが参加可能でしたらレビューの登録時に記載してください。レビューのスケジュールはこちらから確認できます。→ Menu/Review
  2. EasyReviewシステムはメールにてレビュアー（リーダー、CIC、QC-Cornerなど）やマネージャー（PM、PDMなど）にお知らせしますので、エントリー時のレビュアーやマネージャーの記載に誤りが無いようにしてください。
  3. 基本的に開催日の7日前までにエントリーしてください。またレビューの資料は開催日の少なくとも3日前までに提出してください。もし急なレビューが必要な場合は、レビュアーの森下、福田、安部、西、石崎、本田真、赤沼、中村（黒部）、保科、高瀬、長谷に相談してください。
  4. RRのエントリーの際は、プロジェクトの詳細を入力する必要はありません。もしSHOPなどのような情報を示す必要がある場合はタイトルやコメント欄に記載してください。そして、レビュー番号を受け取った後に、プロジェクト名・開発番号・SHOP名を示すことでレビューページの更新をしてください。
  5. もしPDR、CDR、PQR、PSRにエントリーする場合は、必ず開発番号を指定してください。
  6. 開発種別は次のいずれかを指定します。
    1. NEW （新メニューやドライバーの開発など）
    2. VUP （１人月以上の）
    3. MAINTE （１人月未満の）
  7. PSRは以下の時間帯のみの受付となります。
    1. 10:00 - 11:00 （月曜日を除く）
    2. 14:00 - 15:00
    3. 16:00 - 17:00 （WITH開発のみ）
  8. PSRの際はこのリンクのテンプレートを使用して議事録を作成してください。議事録はEasyReviewシステムにアップロードするのとは別に、review@wni.com、cic@wni.com、qc-corner@wni.comにメールしなければいけません。

  2. タイトルフォーマット: <開発項目> [<開発タイプ>]. 例: “EasyReview Apollo V.1.2.1 [VUP].”
